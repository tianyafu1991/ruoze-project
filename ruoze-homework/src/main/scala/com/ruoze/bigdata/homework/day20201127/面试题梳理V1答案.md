# 为什么Kafka的吞吐量很大而且速度很快，如何保证Kafka数据有序
```
吞吐量大而且速度很快的原因：
1.顺序读写
2.零拷贝，通过零拷贝 减少了数据从内核态读到用户态 再从用户态写到内核态的过程
3.分区 分区落在不同的磁盘上 多个磁盘读写性能比单个磁盘高

保证有序：
1.将需要有序的数据打上相同的key
2.topic单分区(生产上不常用)
```
